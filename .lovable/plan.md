# Каталог проектов домов

Раздел `/proekty` с 160 проектами: каркас (40), СИП (30), газобетон (50), кирпич (40). Артикулы `SHAD-01 … SHAD-160`. Цена за проект не показывается — он входит в стоимость строительства.

## Структура маршрутов

```
src/routes/proekty.tsx            — layout (Outlet)
src/routes/proekty.index.tsx      — каталог с фильтрами
src/routes/proekty.$slug.tsx      — детальная страница проекта
```

Ссылка «Проекты домов» добавляется в главное меню (`src/data/navigation.ts`) и в футер.

## Каталог `/proekty`

- Сетка карточек (3 в ряд десктоп / 2 планшет / 1 моб).
- Карточка: фасад-рендер, артикул, название, площадь, этажность, габариты, материал-бейджи, спальни. Бейдж «Проект включён в стоимость».
- Левая колонка фильтров (на моб — bottom-sheet):
  - Материал (мульти): Каркас / СИП / Газобетон / Кирпич
  - Этажность: 1 / 1.5 (мансарда) / 2
  - Площадь: слайдер 60–350 м²
  - Спальни: 1, 2, 3, 4, 5+
  - Особенности (чипы): гараж, терраса, второй свет, котельная, сауна, мастер-спальня
- Сортировка: новизна / площадь ↑↓ / этажность.
- Поиск по артикулу и названию.
- SSR + SEO: title «Проекты домов под ключ — Шадов и партнёры», description, og.

## Детальная страница `/proekty/SHAD-XX`

Секции сверху вниз:
1. Хлебные крошки → Галерея фасадов (4 ракурса) с lightbox.
2. Хедер: H1, артикул, ключевые цифры (площадь общ/жилая, габариты, этажи, спальни, санузлы, высота этажа).
3. Бейдж-карточка «Проект бесплатно входит в стоимость строительства».
4. Поэтажные планы — табы (1 этаж / 2 этаж / мансарда / кровля), изображение + экспликация помещений списком.
5. Фасады — 4 вида.
6. Описание (3–5 абзацев): для кого, особенности планировки, тренды.
7. **Стоимость строительства из разных материалов** — таблица:
   - Каркас под ключ — `площадь × 38 000 ₽` (берём ставки из `src/data/home-prices.ts`, подтверждены пользователем как пример).
   - СИП — `площадь × 32 000 ₽`
   - Газобетон — `площадь × 45 000 ₽`
   - Кирпич — `площадь × 55 000 ₽`
   Под таблицей примечание: «Расчёт ориентировочный. Финальная смета — после выезда инженера, бесплатно.»
8. Блок «Гибкость проекта» — 3 плитки:
   - Внесём любые изменения в проект — бесплатно
   - Построим по вашему готовому проекту
   - Разработаем индивидуальный проект — входит в стоимость
9. CTA-форма «Получить расчёт по проекту SHAD-XX» (переиспользует `EstimateSection`, slug передаётся в скрытое поле).
10. Похожие проекты (4 шт того же материала, близкие по площади).
11. JSON-LD: `Product` + `BreadcrumbList`.

## Данные

`src/data/projects.ts` — массив из 160 объектов:

```ts
type HouseProject = {
  slug: string;               // "SHAD-01"
  title: string;              // "Современный одноэтажный дом 92 м²"
  materials: ("frame"|"sip"|"gas"|"brick")[]; // основной материал(ы)
  primaryMaterial: "frame"|"sip"|"gas"|"brick";
  area: number;               // общая м²
  livingArea: number;
  width: number; depth: number; // м
  floors: 1 | 1.5 | 2;
  bedrooms: number; bathrooms: number;
  features: ("garage"|"terrace"|"boiler"|"sauna"|"second-light"|"master")[];
  description: string;        // 3–5 абзацев
  rooms: { floor: 1|2|"mansard"; name: string; area: number }[];
  images: {
    facadeMain: string;       // главный рендер
    facades: string[];        // 4 фасада
    plans: { floor: 1|2|"mansard"; src: string }[];
  };
};
```

## Изображения (160 проектов × ~7 изображений ≈ 1100 файлов)

Вариант A (подтверждён): генерируем через `imagegen--generate_image`.
- Главный фасад: photoreal AI render, 1280×960, jpg.
- 4 фасада: упрощённые рендеры (С/Ю/В/З).
- Поэтажные планы: схематичные планы (top-down, чистые линии, мебель), 1024×768, png. Помечаем в шаблоне как «концепт».
- Заметная плашка в галерее: «Изображения — концептуальная визуализация. Точные чертежи выдаются после заключения договора и адаптации проекта.»

Это снимает юридический риск, на котором я останавливался.

Чтобы это было выполнимо за разумное число итераций, генерацию делаю партиями по материалу. Если в процессе пойму, что ~1100 генераций слишком долго/дорого по кредитам, остановлюсь и спрошу: оставить по 3 изображения на проект (главный фасад + 1 план + 1 интерьер) вместо 7.

## Прайс-ставки

Беру из `src/data/home-prices.ts`. Сейчас зафиксирую константой `MATERIAL_RATES` в `src/data/projects-pricing.ts`, чтобы менять в одном месте:

```ts
export const MATERIAL_RATES = { frame: 38000, sip: 32000, gas: 45000, brick: 55000 } as const;
```

Если в `home-prices.ts` цифры другие — синхронизирую с ними.

## Порядок реализации (чекпоинты)

1. **Каркас данных и страниц** (без картинок): тип `HouseProject`, 4 сид-проекта (по 1 на материал), маршруты, карточка, детальная, фильтры, расчёт стоимости, навигация, SEO. → можно посмотреть рабочий поток.
2. **Контент-генерация описаний и спецификаций** для всех 160 — собираю заранее заготовленным набором плановых решений (одноэтажные 60–100 м², полутораэтажные 110–160, двухэтажные 170–280).
3. **Генерация изображений** партиями: каркас (40) → СИП (30) → газобетон (50) → кирпич (40). После каждой партии — короткий статус.
4. **Финал**: sitemap, перелинковка с услугами строительства, проверка типов и сборки.

## Технические детали

- Все данные статичны (массив в `src/data/projects.ts`) — без БД. Так быстрее, дешевле и пригодно для SSR/prerender.
- Изображения кладу в `src/assets/projects/SHAD-XX/` и импортирую — Vite оптимизирует. Если общий вес окажется большим, переведу на `lovable-assets` (CDN).
- Цена считается на лету: `Math.round(area * MATERIAL_RATES[m] / 1000) * 1000`.
- Похожие проекты: тот же `primaryMaterial`, фильтр по `|area − targetArea| < 40`, лимит 4.

## Что НЕ делаю

- БД-таблиц, админки для проектов (статичные данные).
- Калькулятора в карточке с выбором опций — только базовая таблица по материалам.
- Реальных архитектурных чертежей с ГОСТ-размерами — изображения концептуальные с дисклеймером.
